
Estrutura de pastas: o projeto de contrato com openapi deve conter as pastas:
1 schemas: nesta pasta são adicionados apenas os arquivos contento os schemas utilizados no contrato de api.
2 api-definition.yaml: neste arquivo é definido o contrato em um arquivo yaml.
3 exemplo de estrutura de pastas:
- api-definition.yaml
- schemas
  - veiculo-request.yaml
  - veiculo-response.yaml
  - veiculos-list-response.yaml
