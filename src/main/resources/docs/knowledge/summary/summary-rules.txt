A resposta para o usuário deve apresentar a lista de ações que deverão ser executadas para cumprir com o objetivo do usuário em json.

Crie um passo a passo de cada ação (step) necessária para atender o pedido do usuário usando as funções disponíveis de forma que a LLM possa executar cada item individualmente.
Crie um passo a passo onde cada step tenha apenas uma ação.
É possível que o projeto esteja vazio ou contenha apenas uma parte dos arquivos necessários para atender à necessidade do usuário. Portanto crie steps que peça para criar
arquivo caso o mesmo não exista.
Um step com a ação de criar arquivo não pode ser o mesmo step que pede para inserir conteúdo no arquivo. É preciso criar steps separados para cada ação.
Cada step deve ter uma única ação e caso alguma informação de outro step seja necessário, o outro step deve ser declarado no dependsOf. Por exemplo, caso seja necessário
gerar um conteúdo em um arquivo, é necessário depender de outro step que tenha pedido para ler o conteúdo do arquivo antes.

stepName: deve ser escrito em camel case. Ele serve para que outra step possa informar que depente de outra step.
stepAction: é um texto indicando a ação a ser executada. As ações podem ser de consulta de dados para ser usado por outra step ou uma ação de geração de conteúdo que será adicionado em um arquivo.
dependsOf: é uma lista de nomes de outros steps. Essa indicação informa quais os outros steps devem executar

Caso arquivos tenham sido criados e referenciados em outros arquivos, crie um step final pedindo para validar cada referencia já pedindo para corrigir, caso necessário


==Exemplos de stepActions:
1. Crie arquivo X.
2. Crie contrato A com parametros Q, W e E.
3. Liste os arquivos do projeto.
4. Leia os dados de um arquivo Y.
5. No arquivo Y, implemente schema de cadastro de X com parametros Q, W e E.
6. No arquivo Y, implemente schema de consulta X com parametros id, Q, W e E.
==

==Exemplos de stepName:
1. nomeDaStep
2. acaoDeCriacao
3. acaoDeListagem
4. consultaDados
==

O formato da resposta da lista de passos tem que ser em formato json seguindo a estrutura a seguir:
[
    {
        "stepName": "adicione aqui um nome para esse passo em camel case",
        "stepAction": "adicione aqui a ação que deve ser executada neste passo",
        "dependsOf": ["adicione aqui os nomes das outras questões que devem ser executadas antes desta, como: questões que pedem criação de arquivo que será usado por esta questão etc"]
    },
]
